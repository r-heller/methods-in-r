# Migration log

Generated by `scripts/migrate.py`. One row per source file.

## Pipeline

1. `scripts/build_manifest.py` — walks `../tutorials/` and `../courses/`,
   emits `_data/manifest.json`. **569 method pages + 80 labs = 649 entries.**
2. `scripts/migrate.py` — copies each entry into `chapters/`, `reporting/`,
   or `shared/`, strips site-only YAML keys, appends the *For Reviewers*
   skeleton to method pages (gated on a fence comment, so re-runs do not
   duplicate). Also copies `course{1..4}/code/*.R` to `shared/code/` and
   `courses/shared/data/` to `shared/data/`.
3. `scripts/build_chapter_indexes.py` — generates one `index.qmd` per
   chapter from the manifest: scope paragraph, methods table, labs table.
4. `scripts/build_reporting.py` — generates `reporting/{strobe,consort,
   tripod-ai,prisma,arrive}.qmd` from the manifest, one method-specific
   skeleton per matching method.
5. `scripts/build_cheatsheets.py` — aggregates the 16 weekly cheatsheets
   into `appendices/E-cheatsheets.qmd`.
6. `scripts/inject_crosslinks.py` — adds idempotent *See also* blocks
   wrapped in `<!-- BEGIN see-also -->` / `<!-- END see-also -->` fences.

Re-running the full pipeline is a no-op apart from minor reformatting.
