# Contributing to Methods in R

Thank you for your interest in contributing.

## How to contribute

1. **Issues:** Report errors, suggest improvements, or request topics via [GitHub Issues](https://github.com/r-heller/methods-in-r/issues).
2. **Pull requests:** Fork the repo, create a feature branch, and submit a PR. All PRs must pass CI checks (HTML render, PDF render, citation verification).
3. **Citation discipline:** Every academic reference must have a verifiable DOI, arXiv ID, or PubMed ID.

## Style

- R code follows `tidyverse` style.
- Figures use the Okabe-Ito colorblind-safe palette.
- Method pages follow the 10-section template (Introduction through Reporting, plus For Reviewers).
- Labs follow the 5-step template (Hypothesis, Visualise, Assumptions, Conduct, Conclude).
